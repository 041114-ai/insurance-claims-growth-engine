# 保险理赔增长运营分析体系

## 项目目标

将C端保险经纪平台的理赔环节从"成本中心"转变为"增长引擎"，通过在理赔场景的精准运营，实现可量化的保费增长。

## 核心业务命题

在理赔履约的**什么场景**下，给**什么类型的用户**，在**什么时机**，以**什么样的策略**推荐**什么产品/权益**，能**最大化该场景下的长期保费收入**（LTV）？

## 关键绩效指标

### 核心北极星指标
- 理赔场景驱动的**增量保费总额**

### 一级拆解指标
- **续保端**：理赔用户**续保率**（绝对百分比提升，pp）
- **增收端**：理赔用户**二次开发成功率**（二开率）
- **增效端**：单位策略成本的保费产出（ROI）

### 二级监控指标
- **过程指标**：策略触达率、用户响应率、加保转化漏斗各环节转化率
- **质量指标**：理赔用户满意度/NPS、理赔履约时效

## 项目阶段

### 第一阶段：基建与洞察（第1个月）
- 完成数据基建与用户分群
- 产出可行动的洞察
- 启动首批实验

### 第二阶段：实验启动与验证（第2-3个月）
- 通过A/B测试验证策略有效性
- 开始产生增量保费

### 第三阶段：规模化与优化（第4-6个月）
- 将成功策略在更广泛人群中推广
- 形成策略矩阵

### 第四阶段：体系化与自动化（第7个月及以后）
- 建成数据驱动的自动化增长闭环
- 持续发现增长机会

## 技术架构

- **数据处理**：Python (pandas, numpy, scikit-learn)
- **分析看板**：Streamlit
- **实验框架**：自定义A/B测试框架
- **部署**：GitHub Pages

```
├── data/             # 数据目录

│   ├── experiment/   # 实验模块

├── docs/             # 文档
├── requirements.txt  # 依赖项
└── README.md         # 项目说明
```

## 快速开始

1. 克隆仓库
2. 安装依赖：`pip install -r requirements.txt`
3. 运行分析脚本：`python scripts/analysis.py`
4. 启动看板：`streamlit run src/dashboard/app.py`
